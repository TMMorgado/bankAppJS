PT:

Este pequeno projeto teve como objetivo aprofundar os meus conhecimentos de JavaScript, em especial no trabalho com arrays e objetos.

Ao longo do desenvolvimento, fui extraindo e aplicando vários métodos de arrays e objetos, integrando-os numa pequena aplicação que simula uma experiência bancária.

Funcionalidades principais

Sistema de login dinâmico
No início da aplicação existe um sistema de login que, consoante as credenciais introduzidas, apresenta diferentes views para cada utilizador, reutilizando o mesmo HTML de base.

Visualização de movimentos
É possível visualizar os diferentes movimentos associados à conta de cada “objeto” (que representa uma pessoa/conta bancária).

Simulação de apagar conta
A aplicação permite simular a eliminação de uma conta, removendo o objeto correspondente e retirando também o respetivo acesso (login e view).

Simulação de transferências bancárias
É possível simular transferências entre diferentes contas, com base no IBAN do destinatário.

Simulação de empréstimos
A app simula pedidos de empréstimo, condicionados pelo valor disponível na conta (saldo).

Ordenação de movimentos
Através do método sort, os movimentos podem ser ordenados de forma crescente ou decrescente.

Contexto

Este projeto foi desenvolvido com base num curso de JavaScript da Udemy.
A aplicação foi inicialmente replicada e, em seguida, modificada e adaptada ao meu estilo, mantendo a estrutura base ensinada no curso.


EN:

This small project was created to deepen my knowledge of JavaScript, with a particular focus on working with arrays and objects.

Throughout the development, I extracted and applied several array and object methods, integrating them into a small application that simulates a simple banking experience.

Main features

Dynamic login system
At the start of the app, there is a login system that, depending on the credentials provided, displays different views for each user while reusing the same base HTML.

Transaction overview
The app allows you to view the different transactions associated with each account “object” (which simulates a person/bank account).

Account deletion simulation
It is possible to simulate deleting an account, which removes the corresponding object and also revokes its access (login and view).

Bank transfer simulation
The application simulates bank transfers between different users based on the recipient’s IBAN.

Loan simulation
The app simulates loan requests that depend on the current account balance.

Sorting transactions
Using the sort method, the list of transactions can be ordered in ascending or descending order.

Context

This project was built as part of a JavaScript course on Udemy.
The application was initially replicated from the course material and then customized and adapted to my own style, while still being based on the original structure.
